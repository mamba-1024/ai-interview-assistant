## AI 面试助手 Chrome 插件 — MVP 设计文档

---

### 一、市场机会

AI 面试技术市场规模在 2023 年为 6.6 亿美元，预计 2030 年将达到 11.2 亿美元。77% 的求职者已在使用 AI 工具辅助求职，近 50% 的求职者用 AI 优化申请材料。然而现有产品在 Chrome 插件形态上的覆盖极为薄弱——真正做到浏览器插件的竞品只有 2 个（InterviewPrep AI 和一个周末项目 HireZen），且功能远不完善。

高价竞品已经验证了付费意愿：Final Round AI 月费高达 60-149 美元，LockedIn AI 月费 35-70 美元。这为中间价位（15-35 美元/月）的高性价比插件留出了巨大的市场空间。

---

### 二、竞品弱点分析（你的突破口）

通过对 Final Round AI、LockedIn AI、OphyAI、InterviewPrep AI 等产品的用户评价分析，现有竞品存在以下核心痛点：

**痛点 1：屏幕共享时被检测到**
这是用户最大的恐惧。多个独立测试显示，LockedIn AI、Parakeet AI 等产品的覆盖层在屏幕共享时对面试官可见。这是被淘汰的直接风险。

**痛点 2：实时响应卡顿**
Final Round AI 用户频繁报告面试过程中出现"卡死"和"延迟"。系统资源占用大的桌面应用在视频通话时尤其不稳定。

**痛点 3：AI 回答过于通用**
用户抱怨 AI 无法有效结合简历中的具体项目经验，生成的回答像模板，缺乏个人化。

**痛点 4：价格高但有限制**
高端工具收费 60-150 美元/月，但仍限制面试次数，令活跃求职者不满。

**痛点 5：安装门槛高**
多数竞品要求安装桌面应用，首次配置复杂。Chrome 插件的轻量化形态反而是差异化优势。

---

### 三、目标用户画像

**核心用户：** 25-35 岁的科技/金融行业求职者，正在进行远程面试，每周 2-5 场面试。他们愿意为求职成功付费，但觉得 100 美元/月太贵。

**次要用户：** 应届毕业生（价格敏感，面试频率低但总量大）、转行者（需要更多准备和信心）、非英语母语求职者（需要语言辅助）。

**定价甜区：** 15-35 美元/月，介于 OphyAI（9 美元）和 Final Round AI（60 美元）之间。

---

### 四、MVP 功能设计（三期路线）

#### 第一期：MVP 核心（4-6 周开发）

只做三件事，但每件事都做到比竞品好：

**功能 1：智能面试准备（Pre-Interview）**
用户上传简历 + 粘贴目标职位 JD，AI 生成：针对该职位的高频面试题库（含行为面试、技术面试、情景面试）、基于用户简历的 STAR 格式答案框架、公司文化和面试风格简报。

为什么先做这个：不依赖实时音频技术，风险最低；行业专家越来越推荐"准备阶段用 AI 才是安全且 ROI 最高的方式"；这个功能本身就可以作为免费获客工具。

**功能 2：实时面试辅助（Real-Time Copilot）**
在 Google Meet / Zoom Web / Teams Web 上运行，通过 Chrome Side Panel 展示：实时转录面试对话、自动识别面试官提问、基于用户简历和 JD 生成个性化回答提示。

关键差异化——解决竞品最大痛点：使用 Chrome Side Panel API（而非内容脚本注入）作为主 UI，不覆盖在视频画面上，屏幕共享时面试官完全看不到；轻量级插件形态比桌面应用占用更少系统资源，减少卡顿。

**功能 3：面试复盘（Post-Interview）**
面试结束后自动生成：面试摘要和关键问答记录、表现分析和改进建议、感谢邮件模板。

#### 第二期：增强（MVP 后 2-4 周）

在 MVP 验证市场需求后，增加：AI 模拟面试（语音对话式，可针对特定职位练习）、简历-JD 匹配度评分和优化建议、多语言支持（重点：中文求职者面外企的英文面试场景）。

#### 第三期：护城河（持续迭代）

建立长期竞争壁垒：面试知识库（积累各行业各公司的面试真题和风格）、个性化 AI 训练（根据用户多场面试数据持续优化回答风格）、求职全流程工具链（简历优化 → 投递追踪 → 面试准备 → 面试辅助 → Offer 谈判）。

### 五、技术架构

#### 整体架构

```
┌─────────────────────────────────────────────────┐
│                Chrome Extension                   │
│  ┌──────────┐  ┌──────────┐  ┌───────────────┐  │
│  │  Popup   │  │Side Panel│  │Content Script │  │
│  │ (控制)   │  │ (主UI)   │  │ (最小注入)    │  │
│  └────┬─────┘  └────┬─────┘  └──────┬────────┘  │
│       │              │               │            │
│  ┌────┴──────────────┴───────────────┴─────────┐  │
│  │          Service Worker (事件路由)           │  │
│  └────────────────────┬────────────────────────┘  │
│                       │                           │
│  ┌────────────────────┴────────────────────────┐  │
│  │     Offscreen Document (音频处理)            │  │
│  │  ┌────────────┐  ┌──────────────────────┐   │  │
│  │  │ tabCapture │  │ getUserMedia (麦克风) │   │  │
│  │  └─────┬──────┘  └──────────┬───────────┘   │  │
│  │        └─────────┬──────────┘               │  │
│  │           AudioWorklet (PCM)                 │  │
│  └──────────────┬──────────────────────────────┘  │
└─────────────────┼────────────────────────────────┘
                  │ WebSocket
         ┌────────┴────────┐
         │   Backend API    │
         │  ┌───────────┐  │
         │  │ Deepgram  │  │  ← 实时语音转文字 (延迟 <300ms)
         │  │  STT      │  │
         │  └─────┬─────┘  │
         │  ┌─────┴─────┐  │
         │  │ LLM 层    │  │  ← GPT-4o / Claude (复杂推理)
         │  │           │  │  ← Gemini Nano (轻量任务,免费)
         │  └───────────┘  │
         └─────────────────┘
```

#### 关键技术选型

**语音转文字（STT）：** 推荐 Deepgram Nova-3，延迟低于 300ms，支持流式 WebSocket，每小时成本约 0.46 美元。MVP 阶段也可先用 Web Speech API（免费）做原型验证。

**大语言模型：** 采用混合策略。Chrome 内置的 Gemini Nano 处理轻量任务（问题分类、快速提示、情感分析），免费且无延迟；GPT-4o 或 Claude 处理复杂任务（生成个性化 STAR 回答、技术问题解答）。

**UI 方案：** 主界面使用 Chrome Side Panel API（约 400px 宽，全高），不注入目标页面 DOM，避免平台更新导致崩溃，也不会在屏幕共享时被看到。仅在 Meet/Zoom/Teams 页面上注入一个最小的浮动状态指示器（"录音中..."）。

**Manifest V3 适配：** Service Worker 有 5 分钟空闲超时限制。解决方案是使用 Offscreen Document 处理持久连接（WebSocket、音频处理），Service Worker 只做事件路由。

#### 每场面试的成本估算

Deepgram STT 约 0.46 美元 + 云端 LLM 约 0.30-0.80 美元 + Gemini Nano 免费 = 总计约 0.76-1.26 美元/小时。按 25 美元/月订阅、用户月均 4 场面试计算，毛利率约 80-87%。

### 六、变现策略

#### 定价模型：Freemium + 订阅制

| 层级 | 价格 | 内容 |
|------|------|------|
| Free | 0 美元 | 面试准备功能（每周 2 次）、简历分析 1 次、无实时辅助 |
| Pro | 19 美元/月 | 无限面试准备、实时辅助（每月 8 场）、面试复盘 |
| Pro Unlimited | 29 美元/月 | 所有功能无限制、模拟面试、优先支持 |
| Annual | 199 美元/年 | 相当于 Pro 的 16.6 美元/月，锁定长期用户 |

#### 为什么这个定价能赢

Final Round AI 的 Interview Copilot 单独收费 60 美元/月，LockedIn AI 的 Pro 版 70 美元/月。你以 19-29 美元/月提供同等甚至更好的体验（更轻量、更安全、更个性化），形成明显的性价比优势。免费层作为获客漏斗，让用户先体验面试准备的 value，再转化为付费用户。

#### 额外收入来源

后期可考虑：B2B 合作（与招聘平台、求职社群合作分发）、Career Coach 版（给职业顾问使用的管理后台）、数据洞察报告（匿名化的面试趋势数据）。

### 七、GTM（Go-to-Market）策略

#### 第一阶段：种子用户（上线前 2 周 - 上线后 1 个月）

在 Reddit 的 r/cscareerquestions、r/jobs，V2EX，即刻，小红书求职板块发布"我做了个 AI 面试工具"的体验帖，提供免费 Pro 试用换取反馈。目标：100 个种子用户。

#### 第二阶段：内容获客（持续）

在 YouTube / B 站 / 小红书发布"AI 面试实测"、"大厂面试准备"等内容，自然植入产品。撰写 SEO 文章覆盖"AI interview assistant"、"Google Meet interview tips"等关键词。

#### 第三阶段：口碑裂变

设计"邀请朋友各得 1 场免费面试"机制。求职者在社群中分享面试经验时自然提到工具。

#### Chrome Web Store 优化

标题包含核心关键词："AI Interview Assistant - Real-time Copilot & Prep"。描述中覆盖：interview copilot, AI interview prep, real-time interview assistant, Google Meet, Zoom interview 等关键词。争取 Chrome Web Store 的 "Featured" 推荐。

### 八、风险与应对

**风险 1：屏幕共享被发现**
应对：使用 Side Panel API 而非覆盖层，完全不注入视频页面 DOM。提供明确的"安全模式"说明，让用户安心。

**风险 2：Chrome Web Store 审核政策**
应对：准备详尽的隐私政策，明确录音功能和数据用途。不存储面试录音，只做实时处理。遵守 Manifest V3 的所有规范。

**风险 3：大厂内置 AI 功能**
应对：Zoom AI Companion、Google 的 AI 功能正在发展，但它们做的是通用会议助手，不会做"面试专用"的深度功能。保持垂直深耕，比通用工具更懂面试场景。

**风险 4：伦理争议**
应对：定位上强调"准备 + 辅助"而非"作弊"。实时辅助提供的是思路和框架提示，而非逐字答案。类比计算器之于数学考试——工具帮助发挥，不替代能力。

**风险 5：音频捕获的技术兼容性**
应对：仅在浏览器内的 Meet/Zoom Web/Teams Web 上运行，不支持桌面应用。明确告知用户使用浏览器版视频工具。提供详细的 setup guide。

### 十、开发时间线

```
第 1-2 周    技术原型（音频捕获 + STT + 基础 UI）
第 3-4 周    面试准备功能（简历解析 + JD 分析 + 题库生成）
第 5-6 周    实时辅助功能（实时转录 + 问题检测 + 回答提示）
第 7 周      面试复盘功能 + 整体联调
第 8 周      内测（50 个种子用户）+ Bug 修复
第 9 周      上线 Chrome Web Store + 启动 GTM
```

总计约 8-9 周可以上线 MVP。
